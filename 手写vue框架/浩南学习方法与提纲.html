个人学习经验

进入正题之前先分享下自己对于框架的一些学习方法（仅供参考，每个人都有自己的学习方法）
我个人把 对于一个框架的掌握程度作为学习的阶段性目标，每一个阶段都有我自己不同的理解

一、 了解😄

  • 目标
    1. 对框架及其生态有基本的了解
    2. 能说出框架的一些思想
    3. 知道大概的使用方法，能够写出 Hello World或 Todo List
  • 如何做？
    1. 熟读文档，对api与使用方法有个基本的了解
    2. 跟着官网自己动手写些简单的demo
    3. 不要将精力放在为什么，能出效果就是成功
    4. 这个阶段不建议直接借鉴他人的项目作为你接触这个框架的第一步，第一印象是非常重要的，你借来的项目对于这个框架的使用并不一定是最优的，跟着官方走，那才是最标准的
    5. 对于angular本人正努力成为这个阶段💔

二、. 熟悉😁

  • 目标
    1. 将上阶段的所有目标实现并有提升
    2. 成为一个合格的api调用者，能够对框架所有的api有了解，提到某个api的时候可以知道是做什么的
    3. 可以使用框架的全家桶
    4. 能够写出实现需求的代码（也许并不是最优的解决方法，但是可以实现就ok）
  • 如何做？
    1. 一定要自己写个项目，只有通过项目才能提升你的使用能力，写demo永远是写demo的水平
    2. 遇到不会的查文档、百度或者问别人都可以，但是一定要记住这个使用方法，下次遇到相似问题的时候不要再被困住了
    3. 这个阶段不要求能写出多优质的代码，可以完成工作中的任务或者实现自己项目中的需求就可以
    4. 对于react本人正努力成为这个阶段😫

三、 掌握😀

  • 目标
    1. 将上阶段的所有目标实现并有提升
    2. 当看到需求的时候可以马上想到解决方法
    3. 能够写出一些高质量的代码
    4. 能够对框架进行二次封装解决业务中的个性化需求
    5. 运行报错的时候可以马上定位到错误的大致范围
  • 如何做？
    1. 不断优化自己目前项目中的代码，多试错
    2. 学习市面上优秀的第三方开源库源码
    3. 尝试接触一些框架的源码
    4. 对于vue本人正努力成为这个阶段🐶

四、 精通😎

  • 目标
    1. 将上阶段的所有目标实现并有提升
    2. 在使用中能够知道框架有哪些“坑”
    3. 熟悉框架源码，完全掌握框架的运行机制，知道大多数api或功能是如何实现的
    4. 要求再严格一些可以造出一个跟框架差不多的轮子
  • 如何做？
    1. 看源码呗...还能怎么办，反正我是没到这个阶段，我也不知道该怎么办😂

我始终相信当你坚持下去就定会有收获，即使最后失败也可以有不一样收获。永远不要急于求成做超出自己能力范围的事情，制定好适合自己计划并脚踏实地的坚持下去。既然我们选择了这个行业就定要保持一颗不断学习的❤️加油吧，骚年们！ --沃-兹吉硕德

进入正题，vue原理

一、俗到家的一句话

百度“vue原理”这个关键词，90%的几率你会看到类似的一句话： “vue使用Object.defineProperty对数据进行getter与setter的转化，但由于ie8及以下的浏览器不支持Object.defineProperty这个属性，所以vue不兼容ie8及以下的浏览器”
那么，Object.defineProperty到底是个什么东西呢？ mdn走起!

  • value：属性对应的值,可以使任意类型的值，默认为undefined
  • writable：属性的值是否可以被重写
  • enumerable：此属性是否可以被枚举
  • configurable：是否可以删除目标属性或是否可以再次修改属性的特性
  • get：获取对象值的时候触发
  • set：设置对象值的时候触发

三、vue的三个核心类

  1. Observer：用来劫持并监听所有属性，如果有变动的，就通知订阅者
  2. Dep：Observer与Watcher的中转站，对依赖进行收集并通知变化
  3. Watcher：可以收到属性的变化通知并执行相应的函数，从而更新视图
